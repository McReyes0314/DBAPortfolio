Process

In this activity, we are tasked with writing the appropriate SQL syntax for each question. Here are the detailed instructions:

    Question 1

  • Create a table named course_t with the following columns:
    - course_id (INTEGER, primary key, not null)
    - course_name (VARCHAR(50), not null)
    - credits (INTEGER, not null)
  • Insert the following data into the course_t table
    - course_id: 101
    - course_name: "Introduction to Databases"
    - credits: 3
  • Update the credits column for the course with course_id 101 to 4.
  • Revoke permission for a user to query the course_t table

    • Question 2

  • Create a table student_t with the following columns:
    - stud_id (INTEGER, Primary Key, NOT NULL)
    - stud_name (VARCHAR(50), NOT NULL)
    - stud_sec (VARCHAR(30))
    - stud_prog (VARCHAR(20))
    - stud_yr (VARCHAR(5))
  • Add a column email (VARCHAR(100)) to the student_t table.
  • Insert the following record into the student_t table:
    - stud_id: 101, stud_name: "Alice", stud_sec: "A", stud_prog: "CS", stud_yr: "1"
  • Write a query to retrieve all records from the student_t table where stud_sec is "A".
  • Delete all records from the student_t table but retain the table structure.
  • Create a table grades_t with the following constraints:
    - grade_id as the Primary Key
    - stud_id as a Foreign Key referencing student_t
  • Update the stud_yr of the student with stud_id 101 to "2".
  • Write the syntax to remove the email column from the student_t table
  • Grant permission to a user to query the student_t table.
  • Revoke permission for a user to delete records from the student_t table.

Learnings

Throughout the activity, I learned the following aspects

  • I learned how to create databases and design tables with specified columns and constraints.
  • I also learned to set default values for columns, and to create, grant, and revoke access permissions for specific users who can access the table.
